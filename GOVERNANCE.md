# Governance

## Project

Factory Intelligence Platform

## Community

Open Factory Initiative

## Current Governance Model

This project is currently maintainer-led while the initial architecture and MVP are established.

## Values

- Open-source collaboration
- Practical manufacturing usefulness
- Safe industrial assumptions
- Clear documentation
- Tested software
- Human-governed AI
- Respectful participation

## Decision Making

Major decisions should be documented in `docs/decisions/`.

Examples of major decisions:

- Architecture changes
- Service boundary changes
- Data contract changes
- Dependency strategy
- Governance/action model
- Security model
- Production integration strategy

## Contributor Path

As the project matures, contributors may become maintainers by consistently providing:

- Quality code
- Useful docs
- Thoughtful reviews
- Manufacturing domain insight
- Tests and examples
- Constructive community support

## Maintainer Responsibilities

Maintainers should:

- Protect safety principles
- Review changes carefully
- Keep the roadmap clear
- Encourage new contributors
- Avoid unnecessary complexity
- Document important decisions
