# Prompt 12: Documentation Hardening

Act as a senior technical writer and open-source maintainer.

Goal: make the repository understandable for new contributors.

Review and update:

- README
- CONTRIBUTING
- DEVELOPMENT
- ARCHITECTURE
- TESTING
- DATA_CONTRACTS
- MVP_SCOPE
- LEARNING_LOG
- API docs

Requirements:

- Remove stale instructions
- Add accurate run/test commands
- Add diagrams if helpful
- Add troubleshooting section
- Ensure docs match code
- Do not change application code unless needed to correct docs
