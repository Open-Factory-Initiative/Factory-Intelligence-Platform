# Prompt 11: End-to-End Tests

Act as a senior QA automation engineer.

Goal: implement the first end-to-end test for the MVP workflow.

Workflow:

1. Start local stack or test harness.
2. Generate gradual drift scenario.
3. Ingest events.
4. Detect drift.
5. Open web UI.
6. View detection detail.
7. Inspect evidence timeline.
8. Approve or reject recommendation.
9. View RCA/CAPA draft.
10. Confirm audit state.

Requirements:

- Use Playwright if UI exists
- Use deterministic simulator seed
- Make test stable
- Document how to run e2e tests
- Update learning log
